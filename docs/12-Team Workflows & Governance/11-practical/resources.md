## 3. Books

Recommended reading for team leads, AI Champions, and developers wanting to deepen their understanding.

| Title | Author(s) | Description | Level |
|-------|-----------|-------------|-------|
| *Prompt Engineering for Developers* | Various | Practical guide to writing effective prompts for code generation, focused on developer workflows. | 🟢 |

| *Thinking, Fast and Slow* | Kahneman | Understanding cognitive biases helps explain why developers over-trust or under-trust AI output. | 🟣 |

## 7. Tools & Extensions

Complementary tools that enhance AI-assisted development workflows.

### Code Quality & Security

| Tool | Description | Level |
|------|-------------|-------|
| [GitHub Advanced Security](https://docs.github.com/en/get-started/learning-about-github/about-github-advanced-security) | CodeQL scanning, secret scanning, and dependency review. Essential quality gate for AI-generated code. | 🟢 |
| [Semgrep](https://semgrep.dev/) | Lightweight static analysis that can catch AI-specific code patterns. Supports custom rules for your team's standards. | 🔵 |
| [Gitleaks](https://github.com/gitleaks/gitleaks) | Secret detection for git repos. Must-have pre-commit hook to catch AI-generated code with hardcoded secrets. | 🟢 |
| [SonarQube / SonarCloud](https://www.sonarsource.com/products/sonarqube/) | Continuous code quality inspection. Useful for tracking quality trends as AI adoption increases. | 🔵 |

### Productivity & Workflow

| Tool | Description | Level |
|------|-------------|-------|
| [pre-commit](https://pre-commit.com/) | Framework for managing pre-commit hooks. Essential for local quality gates before AI-generated code enters version control. | 🟢 |
| [Conventional Commits](https://www.conventionalcommits.org/) | Standardized commit message format. Helps AI generate better commit messages when this convention is documented. | 🟢 |
| [Husky](https://typicode.github.io/husky/) | Git hooks for JavaScript projects. Easy setup for pre-commit and pre-push quality checks. | 🟢 |
| [act](https://github.com/nektos/act) | Run GitHub Actions locally. Test your quality gate workflows without pushing to remote. | 🔵 |

### Context & Knowledge Management

| Tool | Description | Level |
|------|-------------|-------|
| [MCP (Model Context Protocol)](https://modelcontextprotocol.io/) | Open protocol for connecting AI tools to external data sources. Enhances AI context with your team's specific knowledge. | 🟣 |
| [Ollama](https://ollama.com/) | Run open-source AI models locally. Useful for air-gapped environments or privacy-sensitive codebases. | 🟣 |
| [LM Studio](https://lmstudio.ai/) | Desktop app for running local AI models. Alternative to Ollama with a GUI interface. | 🟣 |

## Creating a Team Learning Plan

Use these resources to build a structured learning path for your team:

### Month 1: Foundations (All Team Members)
- [ ] Complete GitHub Copilot fundamentals course
- [ ] Read "Why Team Workflows Matter" and "AI-First Culture" docs
- [ ] Set up tools using the Developer Onboarding Checklist
- [ ] Join 1-2 relevant communities

### Month 2: Practices (All Team Members)
- [ ] Complete a prompt engineering course
- [ ] Read 3-5 blog posts on AI-assisted development
- [ ] Practice with team prompt library
- [ ] Attend first AI retrospective section

### Month 3: Optimization (AI Champions + Leads)
- [ ] Read *Accelerate* for measurement frameworks
- [ ] Watch 2-3 conference talks on AI adoption
- [ ] Review research papers on productivity and security
- [ ] Run first maturity assessment

### Ongoing: Stay Current (Everyone)
- [ ] Subscribe to 2-3 newsletters
- [ ] Watch one conference talk per month
- [ ] Contribute to team prompt library monthly
- [ ] Attend AI Champions community of practice (monthly)
