# Building a Champion Network

> Internal AI advocates who drive adoption from the ground up. Champions are experienced AI users who help others, share learnings, and provide critical feedback to leadership.

---

## What Is a Champion?

A champion is not a manager or an AI expert. A champion is a **practitioner** — a developer who:

- Uses AI tools effectively in their daily work
- Is willing to help teammates learn
- Shares what works (and what doesn't)
- Provides honest feedback to leadership
- Represents their team's needs and concerns

```
Leadership ◄──── Feedback ──── Champion Network
    │                              │
    │                              │
    ▼                              ▼
Strategy, Budget              Mentoring, Demos,
Standards, Tools              Prompt Libraries,
                              Team Adaptation
```

### Core (Expected of All Champions)

| Responsibility | Time Estimate | Frequency |
|---|---|---|
| Mentoring teammates on AI tools | 1-2 hours | Weekly |
| Sharing tips and learnings in team channel | 30 min | Weekly |
| Attending champion network meeting | 30 min | Bi-weekly |
| Reviewing and contributing to prompt library | 1 hour | Monthly |
| Gathering and relaying team feedback | 30 min | Bi-weekly |

| Responsibility | Time Estimate | Frequency |
|---|---|---|

| Running demos or lunch-and-learn sessions | 1-2 hours | Monthly |
| Onboarding new team members to AI tools | 2-4 hours | As needed |

| Testing new AI tool features and capabilities | 1-2 hours | Monthly |

## Selection

### Criteria

Champions should be:

✅ **Volunters** — mandatory assignment kills motivation
✅ **Practitioners** — they use AI tools daily, not just theoretically
✅ **Communicators** — they can explain concepts to different skill levels
✅ **Patient** — they don't judge teammates who are slower to adopt
✅ **Diverse** — mix of seniority levels, tech stacks, and teams

Champions do NOT need to be:
- ❌ The most senior developer
- ❌ AI/ML experts
- ❌ Team leads or managers
- ❌ Available full-time for champion duties

### Selection Process

```
1. Announce the champion program (explain what it is and isn't)
2. Accept volunteers (self-nomination)
3. Ensure one champion per team (if multiple volunteers, discuss)
4. Confirm time commitment with champion's manager
5. Kick off with champion orientation session
```

### Ideal Champion Profile

```
Experience with AI tools:     6+ months of regular use
Technical level:              Mid to senior developer
Communication style:          Patient, shares freely, non-judgmental
Availability:                 10-20% time allocated for champion work
Motivation:                   Genuinely interested, not coerced
```

## Support Structure

### Time Allocation

Champions need **dedicated time**, not "do it when you have a spare moment."

**Recommended:** 10-20% of work time (4-8 hours/week for a full-time developer)

This time is used for:
- Mentoring and pairing sessions
- Creating shared resources (prompts, docs, skills)
- Attending champion meetings
- Staying current with tool updates
- Gathering and processing feedback

### Community Channel

Create a dedicated channel (Slack, Teams, etc.) for champions:

```
#ai-champions (private channel)
├── Pinned: Champion handbook, meeting notes, resource links
├── Threads: Tool updates, tip sharing, problem solving
├── Weekly: "What worked this week" sharing round
└── Monthly: Champion spotlight (featured win or learning)
```

### Regular Meetings

**Bi-weekly Champion Sync (30 minutes)**
```
Agenda:
1. Round-robin: What's working on your team? (10 min)
2. Challenges and solutions sharing (10 min)
3. Updates from leadership (5 min)
4. Action items for next two weeks (5 min)
```

**Quarterly Champion Summit (2 hours)**
```
Agenda:
1. Review: Adoption metrics across teams (20 min)
2. Share: Best practices and innovations (30 min)
3. Discuss: Standards updates needed (30 min)
4. Plan: Next quarter goals and focus areas (30 min)
5. Feedback: What does the champion network need? (10 min)
```

### Access to Latest Tools

Champions should have:
- Early access to new AI tool features
- Ability to test beta capabilities
- Budget for experimentation (time and tools)
- Direct line to tool administrators for technical issues

## The Champion as Feedback Loop

One of the most valuable aspects of a champion network is the **bottom-up feedback channel**:

```
Developer has a problem with AI tools
         │
         ▼
Discusses with team champion
         │
         ▼
Champion aggregates feedback from team
         │
         ▼
Shares in champion network meeting
         │
         ▼
Common themes identified across teams
         │
         ▼
Feedback delivered to leadership with data
         │
         ▼
Leadership acts: policy change, tool update, training
```

### What Champions Surface
- Tool limitations that need workarounds
- Standards that are too rigid or too loose
- Training gaps (what people don't understand)
- Tool configuration issues
- Feature requests for AI tools
- Security concerns from practitioners
- ROI evidence (real time savings, quality improvements)

## Champion Program Structure

### Phase 1: Launch (Month 1)

- [ ] Define champion role and responsibilities
- [ ] Get leadership approval for time allocation
- [ ] Announce program and recruit volunteers
- [ ] Select champions (one per team)
- [ ] Run champion orientation session

### Phase 2: Activate (Month 2-3)

- [ ] Champions complete onboarding with their teams
- [ ] First bi-weekly sync meetings
- [ ] Each champion creates first shared resource (prompt, guide, etc.)
- [ ] Feedback channel established
- [ ] Quick wins shared across teams

### Phase 3: Sustain (Month 4+)

- [ ] Regular meeting cadence established
- [ ] Prompt library contributions growing
- [ ] Cross-team knowledge transfer happening
- [ ] Quarterly summits providing strategic direction
- [ ] Champion role rotating or expanding as needed

## Common Challenges

| Challenge | Solution |
|---|---|
| Champion has no time | Ensure manager approves time allocation; reduce scope if needed |
| Champion burns out | Rotate responsibilities, share the load, celebrate contributions |
| Team doesn't engage | Champion focuses on 1-2 willing devs first, builds from there |
| Champions become gatekeepers | Emphasize enabling, not controlling; multiple champions if needed |
| Network becomes a talking shop | Require tangible outputs (prompts, guides, demos) |
| Champions drift from practice | Ensure champions still code daily; champion role supplements, not replaces |
